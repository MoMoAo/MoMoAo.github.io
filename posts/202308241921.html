Shell编程

GPT回答基础

Shell编程是在Linux和其他类Unix操作系统上进行脚本编程的一种方法。Shell是一个命令行解释器，可以执行用户输入的命令，并且也可以用来编写脚本，用于自动化任务和批处理。下面是Shell编程的基础知识：

  1. Shell脚本文件：Shell脚本是一个纯文本文件，通常以.sh为扩展名。你可以使用任何文本编辑器（如vi、nano、gedit等）来创建Shell脚本文件。

  2. Shebang：在Shell脚本文件的第一行通常指定一个Shebang（解释器指令），用于告诉系统用哪个Shell解释器来执行脚本。例如，#!/bin/bash表示使用Bash解释器执行脚本。

  3. 变量：在Shell中使用变量来存储数据。定义变量时不需要声明类型，直接赋值即可。例如：name="John"，然后可以通过$name来引用变量的值。

  4. 输出：使用echo命令来输出文本或变量的值到终端。例如：echo "Hello, World!"。

  5. 输入：使用read命令从用户获取输入，并将输入保存到变量中。例如：read name。

  6. 条件语句：使用条件语句来根据条件执行不同的代码块。常见的条件语句有if、else、elif。例如：

  7. 循环：使用循环结构来重复执行代码块。常见的循环有for循环和while循环。例如：

  8. 函数：使用函数将一系列命令封装起来，方便在不同地方多次使用。例如：

  9. 注释：使用#符号来添加注释，注释内容将被解释器忽略。

  10. 执行脚本：执行Shell脚本需要给予执行权限，使用chmod +x script.sh命令添加执行权限，然后通过./script.sh来执行脚本。

这些是Shell编程的基础知识，通过学习这些知识，你可以编写简单的Shell脚本来完成各种任务，提高工作效率并简化重复性工作。随着经验的增加，你还可以学习更多高级的Shell编程技巧和工具，以及掌握更多系统管理和自动化的能力。

Shell常见的五种类型

五种类型讲解

在Unix和类Unix系统中，常见的五种Shell类型是Bourne Shell (sh)，Bourne Again Shell (bash)，C Shell (csh)，Korn Shell (ksh)，和 Z Shell (zsh)。这些Shell之间有一些区别，主要涉及以下方面：

  1. 功能和语法：

    • Bourne Shell (sh)：是最早的Unix Shell，功能相对较少，语法相对简单，缺乏一些现代Shell的特性。
    • Bourne Again Shell (bash)：是Bourne Shell的改进版本，是大多数Linux系统的默认Shell。它保留了Bourne Shell的特性，并添加了许多额外的功能和命令，提供更好的用户体验和编程能力。
    • C Shell (csh)：语法与C语言类似，支持用C风格的语法编写Shell脚本。有一些方便的特性，但在脚本编程方面不如bash灵活。
    • Korn Shell (ksh)：由David Korn设计，综合了Bourne Shell和C Shell的特性，提供了更多的编程特性和命令。
    • Z Shell (zsh)：在功能上扩展了bash和ksh，并引入了许多新特性和改进。它强调交互式使用，提供更好的命令补全、主题和插件支持。

  2. 交互式使用：

    • zsh在交互式使用方面较其他Shell更强大，提供更多的便利功能和自定义选项。

  3. 兼容性：

    • bash和sh具有较高的兼容性，几乎在所有Unix系统中都可以找到。
    • csh在某些系统上可能不可用，也不太适合编写复杂的脚本。
    • zsh在大多数现代系统上都可用，但可能需要单独安装。

  4. 默认Shell：

    • 大多数Linux系统使用bash作为默认Shell。
    • macOS使用zsh作为默认Shell。

  5. 可编程性：

    • bash、ksh和zsh都提供强大的编程功能，可以编写复杂的Shell脚本。
    • csh在编程方面相对较弱，更适合作为交互式Shell使用。

总体而言，如果你只是需要一个用于简单任务和交互式使用的Shell，bash或zsh可能是更好的选择。如果你需要更高级的Shell编程功能，可以选择bash、ksh或zsh中的任何一个，具体取决于你对特定功能的需求和个人偏好。

bash和zsh

Zsh（Z Shell）和Bash（Bourne Again Shell）是两种功能强大的Unix Shell，它们在很多方面相似，但也有一些不同之处。下面是它们各自的优缺点：

Zsh的优点：

  1. 更强大的自动补全：Zsh提供更高级的命令补全功能，可以根据命令历史和路径自动补全命令、文件名和选项，极大地提高了工作效率。

  2. 更好的主题和外观：Zsh支持更多的主题和外观定制选项，使Shell界面更具吸引力和个性化。

  3. 更好的拼写纠正：Zsh可以自动纠正常见的拼写错误，减少用户输入错误的机会。

  4. 更强大的别名（alias）功能：Zsh的别名功能更灵活，可以支持更多的选项和参数。

  5. 更高级的历史管理：Zsh允许更复杂的历史记录和管理，包括保存时间戳、删除重复记录等。

  6. 更好的通配符支持：Zsh支持更多的通配符选项，使文件匹配更方便。

Zsh的缺点：

  1. 安装和默认支持：虽然Zsh在大多数现代系统上都可以找到，但有些较旧或特殊的系统可能没有预装Zsh，并且可能需要单独安装。

  2. 资源占用：Zsh相对于Bash来说，可能占用更多的系统资源，尤其在较旧的硬件上可能会感觉较慢。

Bash的优点：

  1. 广泛支持：Bash是大多数Linux发行版和Unix系统的默认Shell，几乎可以在任何Unix系统上找到。

  2. 较低的资源占用：相比于Zsh，Bash可能占用较少的系统资源，特别是在较旧的硬件上运行时更为明显。

  3. 较少的学习曲线：由于Bash是默认Shell，更多的教程和资源可用于学习Bash，适合初学者入门学习。

Bash的缺点：

  1. 较弱的自动补全：虽然Bash也支持命令补全，但它在功能和定制选项上比Zsh弱一些。

  2. 较少的主题和外观选项：相比Zsh，Bash的外观和主题定制选项较少。

  3. 较弱的别名（alias）功能：Bash的别名功能较为简单，不支持一些高级选项。

综合而言，如果你需要一个功能强大且高度可定制的Shell，特别是在日常交互式使用和编程方面，Zsh可能是更好的选择。但如果你只是需要一个简单易用的Shell，并且对于学习Shell编程来说，Bash可能更适合初学者。无论选择哪个Shell，都取决于个人偏好和特定的使用场景。

Linux/Unix 系统默认不会在当前目录中查找可执行文件。所以，为了确保能够正确执行脚本，我们需要提供脚本的完整路径或使用 ./ 来指定当前目录。

source执行脚本

(也可以用于更新一些配置文件，source代表在当前环境下重载配置文件，而不是子shell下，那样影响不到当前的主环境)

等价于 . hello.sh 但不等价于./ hello.sh

使用 source 命令执行脚本和直接执行脚本有一些重要的区别。让我们来看看 source 命令的不同之处：

  1. 运行环境：使用 source 命令（或 . 命令，它们等效）执行脚本时，脚本会在当前 Shell 进程中运行，而不是在一个新的子进程中。这意味着脚本中的任何变量、函数和环境变化都会直接影响当前的 Shell 环境。这对于配置环境变量或执行修改当前 Shell 环境的脚本非常有用。

  2. 路径查找：使用 source 命令执行脚本时，不需要脚本具有可执行权限或者在 PATH 环境变量所定义的目录中。你可以在任何目录下使用 source 命令执行脚本，只要提供脚本的完整路径或相对路径即可。

  3. 退出行为：使用 source 命令执行脚本后，脚本执行结束后不会退出当前的 Shell 环境。脚本中的任何改变仍然会保留在当前的 Shell 中。

查看当前运行进程

ps 是一个用于查看当前系统中正在运行的进程信息的命令。它用于显示系统中运行的进程的状态、资源使用情况和其他相关信息。ps 命令在 Linux/Unix 系统中非常常用，用于监控和管理系统进程。

全局变量是可以在父 Shell 和子 Shell 之间进行通信的一种方式。在父 Shell 中定义的全局变量，在子 Shell 中也可以访问和使用，但需要注意的是，子 Shell 中对全局变量的修改不会直接影响到父 Shell 中的变量。这是因为子 Shell 在运行时会创建一个独立的进程，它继承了父 Shell 的环境变量，但对变量的更改只会影响到当前子 Shell。

系统常用变量

在 Linux/Unix 系统中，有许多常用的系统变量，这些变量用于存储有关系统配置、运行环境和用户信息等的信息。以下是一些常见的系统变量：

  1. PATH：包含可执行文件的目录列表，用于在终端中查找命令。当你输入一个命令时，系统会在 PATH 中指定的目录中查找该命令的可执行文件。

  2. HOME：表示当前用户的主目录路径。通常用于指定用户的默认工作目录。

  3. USER 或 LOGNAME：表示当前登录用户的用户名。

  4. SHELL：表示当前用户所使用的默认 Shell 解释器的路径。

  5. PWD：表示当前工作目录的路径。

  6. PS1：表示命令行提示符的格式。可以通过修改该变量来自定义终端提示符的样式。

  7. PS2：表示命令行续行提示符的格式。当你输入一行命令太长需要换行时，会显示 PS2 提示符。

  8. LANG 和 LC_*：用于设置系统的语言和地区设置。LANG 设置整体语言环境，而 LC_* 变量可以设置特定方面的语言和地区，例如时间格式、货币符号等。

  9. UID 和 EUID：分别表示当前用户的真实用户 ID 和有效用户 ID。通常，UID 是当前用户的真实 ID，而 EUID 可能是不同于 UID 的特殊权限的用户 ID。

  10. TERM：表示当前终端类型。用于告诉系统如何正确地显示字符和颜色。

  11. HOSTNAME：表示当前系统的主机名。

  12. OSTYPE：表示当前操作系统类型，通常是 linux-gnu。
